## 사용자 여정 (Journey Maps)

### 🎯 시나리오 A — "혼자 빠르게 점심 결정" (단일 사용자, 30초)

```
[홈] → [룰렛] → 메뉴 후보 입력 → 룰렛 돌림
  → 결과 마음에 듦 → 카카오맵 외부 링크로 식당 찾으러
```

**총 소요시간**: 30초~1분

### 🎯 시나리오 B — "회사 동료 4명과 점심 결정" (팀 사용, 5분)

```
[홈] → [근처 식당] → 식당 3~5개 후보로 담음
  → 홈으로 돌아오면 "이 5개로 팀 투표 시작" CTA
  → [투표 만들기] → 시간 15분 선택 → "투표 만들고 공유하기"
  → [투표 세션] → 카톡으로 공유
  → 동료들 투표 (실시간 반영)
  → 15분 후 자동 종료 (또는 "지금 결정")
  → [결과 화면] → 메뉴 확정
  → 근처 식당 자동 추천 → "길찾기" 클릭 → 카카오맵
```

**총 소요시간**: 5~15분 (대기 시간 포함)

### 🎯 시나리오 C — "동률 발생 케이스" (5:5 동률, 7분)

```
[투표 세션] → 5:5 동률로 종료
  → 자동으로 결선 시작 (2분)
  → 동료들 다시 투표
  → 또 동률 (예: 1:1)
  → 🎰 자동 룰렛 실행 (1.5초 애니메이션)
  → 메뉴 확정 → 식당 추천 → 길찾기
```

**총 소요시간**: 7~10분

### 🎯 시나리오 D — "처음 가는 동네에서 식사" (단일 출장자, 1분)

```
[홈] → [근처 식당] → 위치 권한 허용
  → 주변 500m 식당 표시
  → 마음에 드는 카테고리 식당 외부 링크 탭
  → 카카오맵에서 길찾기
```

**총 소요시간**: 1~2분

## 화면 상태 전이 (State Transitions)

### 투표 세션 화면 상태

```
status: 'voting'
  ↓ expiresAt 도달 또는 closeNow()
status: 'completed' (단독 1위) ──→ 결과 화면 이동
  또는
status: 'runoff' (동률)
  ↓ runoffExpiresAt 도달
status: 'completed' (결선 단독 1위 또는 룰렛) ──→ 결과 화면 이동
```

### 결과 화면 상태

```
finalMethod === 'vote' → 즉시 메뉴 표시 (애니메이션 X)
finalMethod === 'runoff' → 즉시 메뉴 표시 (애니메이션 X)
finalMethod === 'roulette':
  rouletteRevealed = false
    ↓ 1.5초간 동률 후보 순환 (애니메이션)
  rouletteRevealed = true
    ↓ 메뉴 확정 표시
    ↓ 자동으로 식당 검색 시작
```

### 식당 추천 섹션 상태 (v1.2)

```
restaurants = { kind: 'idle' }
  ↓ rouletteRevealed = true 트리거
restaurants = { kind: 'loading', message: '위치 확인 중…' }
  ↓ getCurrentCoords() 성공
restaurants = { kind: 'loading', message: '근처 김치찌개 식당 검색 중…' }
  ↓ searchRestaurantsByMenu() 성공
restaurants = { kind: 'ok', restaurants: KakaoRestaurant[] }
  또는
restaurants = { kind: 'error', message: '...' } (권한 거부 / API 실패 등)
```

## 외부 링크 (정책 준수)

| 외부 링크 | 정책 근거 |
|---|---|
| 카카오맵 장소 페이지 | "단순 정보 확인 타사 사이트" 허용 |
| 카카오맵 길찾기 | "단순 정보 확인 타사 사이트" 허용 |
| 등록 사이트 (없음) | 점심먹Go 는 등록·결제 외부 위임 X |

→ 미니앱 핵심 기능 (룰렛, 투표, 메뉴 결정) 은 **미니앱 내에서 완결**.
→ 외부 링크는 **결정된 메뉴를 어디서 먹을지** 라는 부가 정보 제공 용도.
